Control AngularJS Only Numbers para PISCYS

Objetivo:Permitir la entrada de solo números decimales,enteros o valores de moneda.

Desarrollado con los mismos criterios de los campos en AS DIRECT con HTML, CSS, AngularJS y Javascript.

Ejemplo básico:
Código: Javascript de Vista dentro del controlador
    $scope.Datos = {
        MontoOne: 1504.00,
        ... 
    }
Código: HTML de Vista
                                                    
1<control-only-numbers id="exampleOne" ng-model="Datos.MontoOne" label="Básico">
2</control-only-numbers>

Las propiedades obligatorias son model e id. El primero representa la relacion de la vista con el controlador Angular.js, la propiedad ng-model. La segunda es un id que identifica al control, es importante en caso que se encuentre mas de un control de este tipo en la vista y una mejora con respecto a los controles de AS DIRECT que automaticamente se genera un objeto en el $scope padre con el id provisto por la propiedad.

la propiedad label genera un etiqueta para el control con el string pasado por valor.

Instancia del Objeto OnlyNumbers en el $Scope

Una de las cuestiones que sucedia en AS DIRECT era la imposibilidad de cambiar un estado dentro del controlador Angular.js, o mejor dicho. Los controles desarrollados no tiene tales opciones. Por ejemplo dependiendo de una configuración se desactive o active un control. Esta fue una mejora a plantear

Como comente anteriormente cada vez que se crea una instancia del control control-only-numbers se génera una instancia de la clase Javascript OnlyNumbers dentro del $scope que contiene el control.

IMPORTANTE! Debido al caracter asincronico de Angular.js puede que mientras se este contruendo la instancia del control se trate de ejecutar un metodo que cambie el estado del mismo; esta daria error. Para ello cualquier sentencia que cambie o interactue con una instancia debe ir dentro del metodo build-complete 
            // Desactiva el control.
            $scope.exampleTwo.Disabled(true);
            

            // Activa el control.
            $scope.exampleTwo.Disabled(false);

Propiedades

required

Define el campo como requerido. Equivale a la propiedad HTML required. Al definir esta propiedad como verdadera cada vez que sea haga foco(focus) y luego se salga(blur) ejecutara un mensaje de error. Cuando un campo es obligatorio tiene un fondo amarrillo palido como AS DIRECT.

1 <control-cbu id="ComponentCBUone" label="CBU" required="true">
2</control-cbu>

setdisabled

Define el campo como desactivado. Equivale a la propiedad HTML disabled. Al definir esta propiedad como verdadera el control toma de color de fondo un color gris y el cursor cambia evitando que se pueda editar el valor.

1<div>
2    <control-cbu id="ComponentCBUfour" label="CBU" setdisabled="true">
3    </control-cbu>
4</div>

cssclass

Define una clase css para el campo. En la estructura interna del control; este esta formado por una serie de divs, un label, spans y un input al que se le escribirá el atributo.

1<div>
2    <control-cbu id="ComponentCBUfive" label="CBU" cssclass="blurControl">
3    </control-cbu>
4</div>
.blurControl{
  background-color: #b7cce680 !important;
  border:2px solid skyblue;
 }

ng-class

Equivale al atributo Angular.js ng-class al cual se le puede pasar por valor un objeto JS que determine una clase css.

1<div>
2    <control-cbu id="ComponentCBUseven" label="CBU">
3    </control-cbu>
4</div>
.redControl{
  background-color: #e6b7cb80 !important;
   border:2px solid rgb(241, 92, 92);
 }

readonly

Define un campo de solo lectura

1<control-only-numbers id="exampleSeven" label="Campo de solo lectura" readonly="true">
2</control-only-numbers>

label-class y input-content-class

El primero define una clase CSS para la etiqueta label. El segundo define una clase tambi&etilde;n para el contenedor del input, es decir la clase afectarã al contenedor pero no al input; para que afecte al input se debe contruir una clase de forma descendente al input, por ejemplo: .MyCustomClass input.
